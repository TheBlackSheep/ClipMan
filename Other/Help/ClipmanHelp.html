ClipMan - the Clipboard Manipulator

Clipman is a clipboard manipulation application that sits in the icon tray on Windows and Linux (and possibly MacOSX).
It offers run-time Object Pascal syntax scripting to manipulate the contents of the clipboard.  Additionally, there are some additional features to load//save images to//from the clipboard in various image formats.

Right-click on the tray clipboard icon gives the following options;



Clear Clipboard - just clears the content currently on the clipboard
Manage Scripts - to create, edit or delete scripts
Run Script - to execute a script

Load Clipboard from file -  to load a file to the clipboard - various file types can be loaded at this time; text files (*.txt, *.csv, *.tab), image formats (*.bmp,  *.jpg, *.png) - if the file isn't one of these but you still select a file, an option to try and load it as a text file is provided - if the contents of the file aren't of a textual format then it probably won't work!

Save Clipboard to a file - to save the current contents of the clipboard to a file - This option provides a facilty to save the text or image that's on the clipboard to a file. If the clipboard content is of a text format then the options to save it will be text file based (i.e. *.txt, *.csv, *.tab) although in theory you can add any extension to the file to be saved. If the clipboard content is an image then the options to save it will be image file based (i.e. *.bmp, *.jpg, *.png, *.xpm)

An additional option to save the image file as reduced color bitmap is provided.  Bitmaps copied to the clipboard from the desktop or active window on Windows are usually quite large and so this feature provides a way of reducing the number of colors in the bitmap to a usable level whilst still retaining sufficiently good color content for things like desktop or active window screenshots- this feature is experimental and is really only related to running the Clipboard Manipulator on Windows - on Linux or OSX the image on a clipboard is likely to be of a PNG or JPG format.

Options - this provides a few configurable options for ClipMan
About - to display the about dialog
Exit - to exit from the application

Managing Scripts



Choose "Add New" or "Edit" - to write a new script or edit existing ones

Delete - to delete the currently selected script - confirmation before deletion is requested.

Help - to show this file (Note many examples of how to use the functionality are included in the sample scripts file - many of the functions are listed here

Close - just closes this window and return to the tray


Adding or Editing Scripts



Scriptname is the name of the script and will be the name you use to select the script from the popupmenu
Category - provides an option to categorize the script so it appears on a sub-menu of the trayicon - leave blank to appear on the main branch of the popupmenu
Description - describe what your function does
ScriptType - reserved for future use
Script - write your script here - if it's a big script then it might be useful to maximize the screen to see it better - see the examples in the initial scripts file and a list of some functions here
Input (Test Data) - enter your test data here - will be setup with the default in the Options if adding a new script
Test button - press this to apply your script to the test data and see the results in the "Output" window - note this doesn't affect the clipboard - the test data is considered to be the contents of the clipboard during testing, if you want to test what's already on the clipboard just paste it into the "Input (Test Data)" area
Output - this is where the test results from applying your script to the test data can be seen - use this to prove your script works correctly

Load Script and Save Script - allow you to load a function from a file into the script editor (of course you could also just copy and paste it from the clipboard if it's there
Help - just shows this file
Cancel - just avoids saving your changes - note: if you have made a change no warning will be given
OK - saves your script to the scripts database

Options

This dialog allows the user to control what start script initially appears in the script editor and the example data in the test area when a new function is being written -  just change the script to whatever skeleton script you're happy with and some initial test data that would be appropriate in the majority of situations.



The checkbox "Show modified contents of clipboard as balloon hint"  will provide a temporary hint over the icon in the tray of the first few lines of the modified clipboard contents after a script has been run.  If this option is unchecked no indicator will be provided.

The spinbutton "Period the hint is displayed (in secs)" is the time in seconds that the temporary hint will be shown if this option is enabled.

Example code and special functions

The scripting language used here is "pascalscript" which is an abridged version of Object Pascal.

Standard pascal functionality is available "type", "const, "var", function and procedure calls and some basic classes are included at this time.  Of particular use is the TStringList class which can be declared and used in the same way that Delphi/Lazarus use this class (note: the "ValueFromIndex" function available in later implementations of the TStringList class is not implemented here).

The Clipboard is copied at runtime to a TStringList instance called "Clippy".  To manipulate the clipboard therefore it is necessary to changes the lines of the Clippy variable applying functions and amendments to each line or the whole text accordingly.

In addition to the various methods and properties available from the TStringList class there are many others included such as IntToStr, FloatToStr, StrToInt, FloatToStr etc that are available as standard.  There are various extra functions that have been added to make the scripting facilities a little easier (see below).


Of course, if a function doesn't exist you could add this to your script and still use it in the code, you'd just have to have it as a declared function on each script where you want to use it prior to your main begin/end. pair.  There are many examples of this in the test/examples and utility scripts included in the base script library.

Various "special" functions are available, these are;
  • function ShowSaveDialog:string; - provides an option to get the name of a file to save something at runtime, note: you'll need to handle an empty return here!
  • function ShowOpenDialog:string; - gets the name of a file to open a runtime - note: you may need to handle any file issues here
  • function IfFileExists(Filename:String):Boolean; - checks whether a file exists or not!
  • procedure ShowMessage(Msg:string); - just displays a message to the user
  • procedure ErrorMessage(Msg:string); - provides a way of displaying an error message to the user
  • function IsConfirmed(Msg:string):boolean; - use this to get a yes/no option from the user - note: "no" is assumed if dialog is closed
  • function InputBox(Title, Msg, DefaultText :String):string; - use this to get some runtime input from the user
Examples of most of these functions are included in the basic scripts library.  These could be removed to free up some space on the menu once you are comfortable with the application and what it can do.

If a particular function is worth adding to the core library then let me know and I'll see if it can be incorporated - I had intentions to include some RegExpr functions but at this time this is not incorporated in the library.

Future functionality may include functions to manipulate other clipboard types such as bitmaps or metadata - but that's for another day!
